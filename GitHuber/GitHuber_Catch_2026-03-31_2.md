> **One-Line Pitch:** A Claude Code plugin that brings OpenAI's Codex capabilities directly into your existing development workflow for code reviews and task delegation.

> [!summary] 📦 1. Core Identity
> At its core, this is a bridge between two powerful AI coding assistants - OpenAI's Codex and Anthropic's Claude Code. It's not a standalone tool but rather a plugin that extends Claude Code's functionality by allowing developers to leverage Codex's specialized code analysis capabilities without ever leaving their current environment. The architecture is straightforward: it acts as a middleware that translates Claude Code's slash commands into API calls to OpenAI's Codex service, then returns the results back to the Claude interface. This solves a significant pain point for developers who might prefer Claude's workflow but occasionally need Codex's unique strengths, particularly its deep understanding of code patterns and its adversarial review capabilities that can challenge design decisions rather than just pointing out syntax errors.

> [!example] 💎 2. The "Lobster" Value
> The real value here lies in its ability to provide two distinct review modes that most other tools can't match: the standard `/codex:review` for comprehensive analysis and the `/codex:adversarial-review` that actively challenges your design choices and assumptions. This adversarial mode is particularly valuable for high-stakes code where you need to identify potential failure modes, race conditions, or alternative approaches that might be safer or more efficient. Commercially, this plugin eliminates the need for developers to context-switch between different tools or maintain separate subscriptions if they already use Claude Code and have access to OpenAI's services (including the free tier). The ability to delegate background tasks through `/codex:rescue` also means you can offload time-consuming code analysis without leaving your editor, potentially saving hours of manual review time each week.

> [!quote] 🛠️ 3. Deployment & Friction
> The deployment process is surprisingly smooth for a plugin that bridges two different AI ecosystems, though it does have a few prerequisites. The setup requires Node.js 18.18+ and either a ChatGPT subscription or OpenAI API key - no surprises there. The installation is as simple as adding the marketplace and running three commands: `/plugin marketplace add openai/codex-plugin-cc`, `/plugin install codex@openai-codex`, and `/reload-plugins`, followed by `/codex:setup` which conveniently handles the Codex installation if npm is available. The only friction point comes when you need to authenticate with OpenAI's services via `!codex login`, but this is a one-time setup. For best results, make sure you have the OpenAI CLI installed globally (`npm install -g @openai/codex`) before starting the setup process to avoid any authentication hiccups.

> [!abstract] ⚙️ 4. The Ideal Workflow & SOP
> This plugin shines brightest in scenarios where you're working on complex, multi-file changes that require both comprehensive review and adversarial challenge. Imagine you're implementing a critical authentication system - you could first run `/codex:review --base main` in the background to get a general analysis, then follow up with `/codex:adversarial-review "focus on potential security vulnerabilities and edge cases in the auth flow"` to stress-test your implementation. For particularly stubborn bugs, you could delegate investigation with `/codex:rescue "trace the root cause of this intermittent failure in the payment processing module"`. The plugin fits perfectly into a modern AI/Dev pipeline by serving as a quality gate before code merges, catching issues that might be missed by automated linters but are too nuanced for static analysis. Its background processing capabilities also mean you can queue up multiple reviews while continuing to work, making it ideal for teams practicing continuous code review without context switching.
